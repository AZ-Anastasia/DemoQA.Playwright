## Создание проекта
Создать папку для решения
```
mkdir Playwright
cd Playwright
```

Создание пустого проекта:
```
dotnet new sln -n DemoQA.Playwright
```

Создание проекта для тестов:
```
dotnet new nunit -n DemoQA.Tests
```

Создание проекта для страниц и элементов:
```
dotnet new classlib -n DemoQA.PagesAndControls
```

Добавление ссылки в проект:
```
dotnet add DemoQA.Tests reference DemoQA.PagesAndControls
```

Настройки для Playwright в PlaywrightSetup.cs

Как настраивать GitHub Actions: \
https://learn.microsoft.com/ru-ru/dotnet/devops/github-actions-overview

https://learn.microsoft.com/ru-ru/dotnet/devops/dotnet-test-github-action

Про настройку .yaml/.yml: \
https://github.com/actions/setup-dotnet

Про настройку Allure:\
https://github.com/simple-elf/allure-report-action

## Задачи для автотеста на demoqa.com

### TestID 001

#### Описание
Автоматизировать проверку количества карточек элементов на странице «Elements»

#### Предусловие
Пользователь находится на сайте `https://demoqa.com/`

#### Шаги для теста:
1. Проверить, количество карточек элементов на странице равно `6`


### TestID 002

#### Описание
Автоматизировать выбор и проверку выбранного Radio Button через форму на странице «Elements» → «Radio Button»

#### Предусловие
Пользователь находится на сайте `https://demoqa.com/`, находится в разделе `Elements` → `Radio Button`

#### Шаги для теста:
1. Проверить, развернута ли вкладка `Elements`
   1. Если вкладка закрыта, нажать на нее
2. Проверить, что текст заголовка совпадает с текстом выбранного элемента
3. Нажать на радио кнопку с текстом `Impressive`
4. Проверить, что эта кнопка выбрана
5. Проверить, что текст ниже содержит текст кнопки - `Impressive`


### TestID 003

#### Описание
Автоматизировать добавление информации о новом пользователе через форму на странице «Elements» → «Text Box»

#### Предусловие
Пользователь находится на сайте `https://demoqa.com/`, находится в разделе `Elements` → `Text Box`

#### Шаги для теста:
1. Проверить, развернута ли вкладка `Elements`
   1. Если вкладка закрыта, нажать на нее
2. Проверить, что текст заголовка совпадает с текстом выбранного элемента
3. Заполнить поля формы:
- `Full Name` (например, Иван Иванов)
- `Email` (например, ivan.ivanov@example.com)
- `Current Address` (например, г. Москва, ул. Примерная, д. 1)
- `Permanent Address` (например, г. Москва, ул. Примерная, д. 1)
4. Нажать кнопку `Submit`
5. Проверить, что введённые данные корректно отображаются в блоке ниже формы

### TestID 006

#### Описание
Автоматизировать открытие новой вкладки на странице Alerts, Frame & Windows» → «Browser Windows»

#### Предусловие
Пользователь находится на сайте `https://demoqa.com/`, находится в разделе `Elements` → `Web Tables`

#### Шаги для теста:
1. Проверить, развернута ли вкладка `Elements`
   1. Если вкладка закрыта, нажать на нее
2. Проверить, что текст заголовка совпадает с текстом выбранного элемента
3. Проверить, что в отобразившейся на странице таблице `7` заголовкой (названий столбцов) и `3` строки с данными
4. Проверить, что в таблице в заголовке есть столбец с названием `First Name`
5. Ввести в поле для поиска по таблице имя `Alden`
6. Проверить, что после ввода имени в таблице отображается только строка с данными по введенному имени
7. Нажать кнопку `Edit` в строке с именем `Alden`.
8. Проверить, что появилось модальное окно
9. Изменить фамилию на `Smith`
10. Нажать кнопку `Submit`
11. Проверить, что фамилия в таблице изменилась на `Smith`
12. Нажать кнопку `Delete` в строке с именем `Alden Smith`
13. Проверить, что в таблице не осталось строк по поиску с введенным именем
14. Очистить поле поиска
15. Проверить, что в таблице снова отображаются оставшиеся строки с данными
